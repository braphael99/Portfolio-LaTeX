\section{Standard 15 - Analyzing Code III: Writing down recurrences}
\subsection{Problem \ref{prob}}\begin{required}\label{prob}
Write down a recurrence for the runtime complexity of this algorithm. Clearly justify your answer. You are \textbf{not} being asked to solve the recurrence.

\begin{verbatim}
foo(n) {
    if (n <= 2) {
        return 0
    }

    foo(n/4)
    foo(n/4)
    foo(n/4)
    foo(n/4)
    for (j = 1; j < n; j*4) {
        print()
    }
}
\end{verbatim}


\end{required}
% Either type your answer in below, or uncomment the \includegraphics command
% and use it to insert an approprate image. Try experimenting with the scale
% 0.9 the width option to resize your image if necessary.

%\includegraphics[width=0.9\textwidth]{solution.jpg}

\begin{proof}[Answer]
\[
T_{n} = \begin{cases} \Theta(1)& : n \leq 2, \\
	4T(n/4) + \Theta(n)& : n > 1.
\end{cases}
\]

We start with our base case which is if $n \leq 2$, we just return. This gives us a runtime of $\Theta(1)$. Next we dig into the recursive part of the code. We are recursively calling foo 4 times and dividing n by 4 each time we do so, then we add the runtime of the non-recursive code on the end. This gives us $4T(n/4) + \Theta(n)$.\\
\end{proof}
